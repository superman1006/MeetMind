4 · BaseAgent.process —— tool_calls 循环细节

不再预先注入 RAG，而是把 rag_search_<agent> 工具绑定到 LLM， 由模型自主决定调几次。

sequenceDiagram
    autonumber
    participant Node as graph._node
    participant Agent as BaseAgent.process
    participant Scrub as _scrub_surrogates
    participant Rag as RAGRetriever
    participant LLM as ChatOpenAI bind_tools
    participant Tool as rag_search_AGENT
    participant Chroma as chromadb.Collection

    Node->>Agent: process requirement, history
    Agent->>Scrub: 清理输入中的 surrogate 字符
    Agent->>Rag: reset_tracking
    Agent->>Rag: as_langchain_tool 返回 Tool
    Agent->>LLM: bind_tools tools
    loop 最多 _MAX_TOOL_ITERATIONS=5 轮
        Agent->>LLM: invoke messages
        LLM-->>Agent: AIMessage 可能含 tool_calls
        alt 有 tool_calls
            Agent->>Tool: invoke query
            Tool->>Chroma: query query_texts 取 top_k=3
            Chroma-->>Tool: docs / metadatas / distances
            Tool-->>Agent: 格式化上下文文本
            Agent->>Agent: 追加 ToolMessage content+tool_call_id
        else 无 tool_calls
            Agent->>Agent: final_text 取 ai_msg.content
            Note over Agent: 跳出循环
        end
    end
    Agent->>Agent: _parse_routing 返回 next_agent, done
    Agent-->>Node: AgentResponse message, output_role, done, used_rag, rag_sources

5 · 模块详情（关键文件 / 函数 / 参数）

每个卡片对应一个 .py 文件，列出关键函数签名与一句话用途。

main.py （项目根入口） cli · entry

最薄的入口文件。只把 cli.main:main() 导进来并调用。

  • main() → None

meetmind/agents/base.py agents · core

公共能力：绑定 LLM、tool_calls 循环、路由解析、UTF-8 清理。

  • 　・process(requirement: str, conversation_history: str) → AgentResponse

  • _scrub_surrogates(text: str) → str # 防 httpx 编码崩

meetmind/agents/{architect,backend,frontend,tester,pm}.py agents · 5 roles

每个角色 = BaseAgent 子类 + 一段专属 system_prompt。

  • ArchitectAgent() # 拆需求、分派、汇总，可输出 [DONE]
  • BackendAgent() # 接口/数据模型/性能/安全
  • FrontendAgent() # 页面结构/交互/接口契约
  • TesterAgent() # 测试策略/用例/覆盖率
  • PMAgent() # 用户故事/优先级/验收
meetmind/graph/state.py graph · state

LangGraph 全局共享状态的 TypedDict 声明。

  • class MessageTurn(agent_name, role, message, output_role)
  • class AgentState(requirement, messages[+], next_agent, complete, iteration)
  • # messages 使用 operator.add 作为 reducer，追加合并
meetmind/graph/routing.py graph · routing

所有节点共用的条件边路由函数；按优先级 4 条规则决定下一节点。

  • route_next(state: AgentState) → str
  • ① iteration ≥ max_iterations → END
  • ② state.complete == True → END
  • ③ next_agent ∈ AGENT_NAMES → "<agent>_node"
  • ④ 兜底 → "architect_node"

把 Agent 实例包装成 LangGraph node，组装出最终的 StateGraph。

  • build_agent_graph() → CompiledStateGraph

  • _format_history(messages: list[MessageTurn]) → str
  • _make_node_fn(agent: BaseAgent) → Callable[[AgentState], dict]
  • 　└─ _node(state: AgentState) → dict

6 · AgentState 字段表（图共享状态）

每个图节点 return 的 dict 按字段合并进 AgentState； messages 用 operator.add reducer 追加，其他字段直接覆盖。

字段 类型 合并语义 含义
requirement str 覆盖 架构师本轮输入的原始需求；整轮不变
messages list[MessageTurn] 追加 (operator.add) 仅追加的讨论历史，每个节点 append 一条
next_agent str | None 覆盖 下一个发言 agent 的名字（由 _parse_routing 写入）
complete bool 覆盖 架构师宣布完成（输出 [DONE]）后置 True，触发 END
iteration int 覆盖 当前轮次；≥ max_iterations 时强制结束（防死循环）
